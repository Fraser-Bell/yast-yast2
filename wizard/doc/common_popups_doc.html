ContinueCancelHeadlinePopup (symbol|string headline, string message) -> boolean

Dialog which displays the "message" and has a Continue and a Cancel button. This popup should be used to confirm possibly dangerous actions and if it's useful to display a short headline (without headline ContinueCancelPopup() can be used). The default button is Continue. Returns true if Continue is clicked.

Parameters:
headline short headline or `noheadline
message message string
Example:
 ContinueCancelHeadlinePopup ( "Short Header", "Going on with action ....?" );
See:
ContinueCancelPopup() YesNoPopup() AnyQuestionPopup()

ContinueCancelPopup (string message) -> boolean

Dialog which displays the "message" and has a Continue and a Cancel button. This popup should be used to confirm possibly dangerous actions. The default button is Continue. Returns true if Continue is clicked.

Parameters:
message message string
Example:
 ContinueCancelPopup ( "Please insert required CD-ROM." );
See:
AnyQuestionPopup()

YesNoHeadlinePopup(symbol|string headline, string message) -> boolean

This dialog displays "message" (a question) and has a Yes and a No button. It should be used for decisions about two about equivalent paths, not for simple confirmation - use "ContinueCancel()" for those. A short headline can be displayed (without headline you can use YesNoPopup()). The default button is Yes. Returns true if Yes is clicked.

Parameters:
headline short headline or `noheadline
message message string
Example:
  YesNoHeadlinePopup ( "Resize Windows Partition?", "... explanation of dangers ..." );
See:
YesNoPopup() AnyQuestionPopup()

YesNoPopup(string message) -> boolean

This dialog displays "message" (a question) and has a Yes and a No button. It should be used for decisions about two about equivalent paths, not for simple confirmation - use "ContinueCancel()" for those. The default button is Yes. Returns true if Yes is clicked.

Parameters:
message message string
Example:
  YesNoPopup ( "Create a backup of the config files?" );
See:
YesNoHeadlinePopup() ContinueCancelPopup() AnyQuestionPopup()

LongTextPopup( string headline, term richtext, integer hdim, integer vdim )

This dialog displays large amounts of text in rich text format that might need scrolling. The text background color is a very light grey.

Parameters:
headline short headline
richtext text input is `Richtext()
hdim initial horizontal dimension of the popup
vdim initial vertical dimension of the popup
Example:
 LongTextPopup ( "Package description", `Richtext("
Hello, this is a long description .....
"), 50, 20 );

ConfirmAbortPopup( symbol how_to ) -> boolean

This dialog is used if "Abort" button in the installation dialogs is clicked. The popup shows a text (according to argument "how_to") and the buttons Abort Installation and Continue Installation. Returns true if Abort Installation is clicked.

Parameters:
how_to `painless, `incomplete, `unusable
Example:
 ConfirmAbortPopup ( `painless );

MessagePopup(string message)

A small dialog which shows the "message". The dialog is closed when user confirms with click on OK button. It should be used to display any user information.

Parameters:
message message string
Example:
  MessagePopup("This is an information about ... ." );

WarningPopup(string message)

A popup with headline 'Warning' and the "message". The dialog is closed when user confirms with click on OK button. It should be used to display an important user information (e.g. if something goes wrong, changes are made).

Parameters:
message warning message string
Example:
 WarningPopup("Something is wrong. Please check your configuration." );

NotifyPopup(string message)

A popup with headline 'Notify' and the "message". The dialog is closed when user confirms with click on OK button. It should be used to inform the user about something.

Parameters:
message notify message string
Example:
  NotifyPopup("Your printer is ready for use." );

ErrorPopup(string message)

A popup with headline 'Error' and the "message". The dialog is closed when user confirms with click on OKy button. It should be used to inform about an error.

Parameters:
message error message string
Example:
  ErrorPopup("The configuration was not succesful." );

TimedMessagePopup(string message, integer seconds)

Displays a message with timeout to the user in a small dialog with an OK button.

Parameters:
message string
seconds time in seconds
Example:
 TimedMessagePopup("This is a timed message", 2 );

AnyQuestionPopup( symbol|string headline, string message, string yes_button_message, string no_button_message, symbol focus ) -> boolean

A popup with the "headline", the "message" and two buttons (labeled according the arguments). Returns true if affirmative button is clicked.

Parameters:
headline `notify, `warning, `noheadline or string
message message string
yes_button_message label on affirmative buttons (on left side)
no_button_message label on negating button (on right side)
focus `focus_no, `focus_yes (which button shall have the focus?)
Example:
 AnyQuestionPopup(`warning, "Do really want to ...?", "Install", "Don't do it", `focus_no );

ModuleError(string text) -> symbol

Popup indicates that there is an error in called module. Returns `id of the button which is clicked.

Parameters:
text string
Return value:
`back, `again, `cancel, `next
Example: 
 ModuleError( "The module " + symbolof(argterm)) + " does not work." );
